요소

요소 만들기 열림태그('<>')와 닫힘태그('<\/>')를 이용해 원하는 영역을 감싼다.

포함된 요소(Nesting elements)

요소 안에 다른 요소가 들어갈 수 있다.

My cat is very grumpy.

단 요소가 중첩으로 포개어(overlap or stacking) 지거나, 엇갈려 겹침) 되어있으면 잘못된 문장이다.

My cat is very grumpy.

위의 잘못된 예제와 같이 내포가 제대로 되지 않았을 경우엔 웹 브라우저가 임의로 결과를 보일 것이다. 잘못 된 구문에도 브라우저가 문벱에 흐물흐물(permittable)해서 웬만한 오류는 자체 수정해서 브라우저 창에 랜더링해주기 때문이다. '웬만한 오류는 알아서 처리해 준다.'는 부분에서 초보자는 특히 주의해야 한다.

블럭 레벨 요소 vs 인라인 요소

  • 블럭 레벨 요소(Block-level elements)는 웹페이지 상에 Block을 만드는 요소이다. 블록 레벨 요소는 앞뒤 요소 사이에 새로운 줄(Line)을 만들어냅니다. 즉, 블록 레벨 요소 이전과 이후 요소사이의 줄을 바꾼다. 블록 레벨 요소는 일반적으로 페이지의 구조적 요소를 나타낼 때 사용된다. 예를들어 단락(Paragraphs), 목록(lists), 네비게이션 메뉴(Navigation menu), 꼬리말(Footers) 등을 표현할 수 있다. 블록 레벨 요소는 다른 블록 레벨 요소에 중첩될 수 있다.
  • 인라인 요소(Inline elements)는 항상 블록 레벨 요소내에 포함되어 있다. 인라인 요소는 문서의 한 단락같은 큰 범위에는 적용될 수 없고 문장, 단어 같은 작은 부분에 대해서만 적용될 수 있다. 인라인 요소는 새로운 줄(Line)을 만들지 않는다. 즉 인라인 요소를 작성하면 그것을 작성한 단락내에 나타나게 된다.

속성(Attributes)

속성은 요소에 실제론 나타내고 싶지 않지만 추가적인 내용을 담고 싶을 때 사용한다. 요소는 다음 내용을 지켜야 한다.

  1. 요소 이름 다음에 바로 오는 속성은 이름과 속성 사이에 공백이 있어야 되고, 하나 이상의 속성들이 있는 경우엔 속성 사이에 공백이 있어야 한다.
  2. 속성 이름 다음엔 등호(=)가 붙는다.
  3. 속성 값은 열고 닫는 따옴표로 감싸야 한다.

참과 거짓 속성(Boolean attributes)

값이 없는 속성도 허용된다. 이를 불 속성이라고 하며, 일반적으로 그 속성의 이름과 동일한 하나의 값만 가질 수 있다. 예를 들어 'disavled' 속성을 양식 입력 요소에 할당하면 사용자가 데이터를 입력할 수 없도록 비활성화(회색으로 표시) 할 수 있다.

속성값의 따옴표 생략

어떤 상황에선 속성값에 따옴표 생략이 허용되지만, 다른 상황에서는 마크업을 망쳐버린다. 예상치 못한 오류를 발생시키지 않으려면 속성값에 항상 따옴표를 붙이라고 조언한다.

HTML 문서의 구조

My test page

This is my page
  1. '!DOCTYPE html태그' : 문서 형식을 나타낸다. HTML초창기에 doctype은 HTML페이지가 자동 오류 검사나 다른 유용한 것이 가능한 좋은 HTML을 의미했다. 하지만 유물일 뿐.
  2. 'html태그' : html요소는 전체 페이지의 콘텐프를 포함하며, 기본 요소로도 알려져 있다.
  3. 'head태그' : head 요소는 홈페이지 이용자에게 보이지 않지만 검색 결과에 노출 될 키워드, 홈페이지 설명, CSS 스타일, character setdeclaration 등 HTML 페이지의 모든 내용을 담고 있다.
  4. 'meta charset="utf-8" 태그' : 이 요소는 HTML 문서의 문자 인코딩 설정을 UTF-8로 지정하는 것이며 예시에서 지정한 UTF-8에는 전세계에서 사용되는 언어에 대한 대부분의 문자가 포함된다. 이 설정을 사용하면 페이지에 포함된 모든 텍스트 내용을 처리 할 수 있다. 이것을 설정하면 나중에 문자 인코딩과 관련된 일부 문제를 피하는 데 도움이 될 수 있으므로 문자 인코딩 설정을 하지 않을 이유가 없다.
  5. 'title태그' : title태그는 페이지 제목이 설정되며 페이지가 로드되는 브라우저 탭에 표시되는 제목으로 사용된다. 페이지 제목은 페이지가 책갈피 될 때 페이지를 설명하는 데에도 사용된다.
  6. 'body태그' : body태그는 텍스트, 이미지, 비디오, 게임, 재생 가능한 오디오 트랙 등을 비롯하여 페이지에 표시되는 모든 콘텐츠가 포함된다.

HTML 공백

HTML 요소 내용 내에서 얼마나 많은 공백을 사용하든 (하나 이상의 공백 문자를 포함 하거나 줄 바꿈 포함) HTML 파서는 코드를 렌더링 할 때 각 공백 시퀀스를 단일 공백으로 줄입니다.

Entity references: HTML에 특수 문자 포함

HTML에서 문자 '<', '>', '"', '&'는 특수문자이다. 이들은 HTML 구문 자체의 일부이다. HTML 코드로 해석되지 않게 특수문자를 사용하려면 HTML 특수문자표를 검색해서 사용!

HTML 주석

HTML에는 코드에 주석을 작성하는 메커니즘이 있다. 브라우저는 주석을 무시하여 사용자가 주석을 보지 못하게한다.
